# Compliance Standards

## Applies when

Collecting user data, processing payments, building algorithms, or designing interfaces.

## Checklist

### Privacy by Design
- [ ] Data minimization (collect only needed); purpose limitation; consent management (track + withdraw)
- [ ] Data retention (delete when no longer needed); privacy defaults (most private)
- [ ] Clear, accessible privacy policy; cookie consent banner (GDPR-compliant)

### GDPR
- [ ] Right to access/rectify/erase/port (JSON, CSV); restrict processing
- [ ] Data breach notification within 72h; DPIA for high-risk processing
- [ ] Legal basis for processing (consent, contract, legal obligation, legitimate interest)

### CCPA
- [ ] Right to know/delete/opt-out; "Do Not Sell My Personal Info" link
- [ ] Non-discrimination for exercising rights; service provider agreements

### Accessibility Beyond WCAG
- [ ] Cognitive accessibility (clear language, consistent nav, error prevention)
- [ ] Neurodiversity support (ADHD, autism, dyslexia); motion sensitivity
- [ ] Screen reader testing (VoiceOver, NVDA, JAWS); keyboard-only testing

### Algorithmic Fairness
- [ ] Bias detection (race, gender, age); fairness metrics (equal opportunity, demographic parity)
- [ ] Explainability (why algorithm made decision); human oversight; appeals process; regular audits

### Ethical Design
- [ ] No dark patterns; transparent pricing; easy cancellation (as easy as subscribe)
- [ ] No notification spam; addiction prevention (no infinite scroll unless justified); user wellbeing

### Legal
- [ ] Clear ToS; COPPA compliance (<13); industry regs (HIPAA, PCI-DSS, SOX, FERPA)
- [ ] Export controls; accessibility laws (ADA, Section 508, EN 301 549)
